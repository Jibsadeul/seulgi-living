# regions 백엔드 명세

## 목적

regions 서비스는 지역 선택 UI에서 사용하는 시/도와 시/군/구 데이터를 조회한다.

## 데이터 규칙

| 테이블  | 컬럼    | 타입        | 설명          |
| ------- | ------- | ----------- | ------------- |
| sido    | id      | varchar(2)  | 시/도 ID      |
| sido    | name    | varchar(50) | 시/도 이름    |
| sigungu | id      | varchar(5)  | 시/군/구 ID   |
| sigungu | sido_id | varchar(2)  | `sido.id` FK  |
| sigungu | name    | varchar(50) | 시/군/구 이름 |

## 검증 규칙

- 시/도 목록은 이름 오름차순으로 반환한다.
- 시/군/구 목록은 `sidoId`를 기준으로 필터링한다.
- 존재하지 않는 `sidoId`로 시/군/구 목록을 요청하면 404 오류를 반환한다.
- 시/군/구 목록은 이름 오름차순으로 반환한다.

## 검증 기준

- 시/도 응답만으로 시/도 셀렉트 박스 옵션을 구성할 수 있다.
- 시/군/구 응답만으로 시/군/구 셀렉트 박스 옵션을 구성할 수 있다.
- 존재하지 않는 `sidoId` 요청 시 404 오류를 반환한다.
