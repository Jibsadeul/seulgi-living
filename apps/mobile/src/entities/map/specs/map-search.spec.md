# map-search.spec.md

> 편의시설 지도 — 장소 검색 및 결과 리스트 기능 명세
> 작성일: 2026-06-15

---

## 1. 기능 정의

### 목적

GPS를 허용하지 않은 유저도 동네를 검색어에 포함하면 해당 지역 기준으로 장소를 찾을 수 있어야 한다.
검색 및 카테고리 결과는 지도 마커와 바텀시트 리스트로 동시에 표시한다.

### 핵심 유스케이스

- `빨래방` 검색 → 현재 지도 중심 기준 결과 마커 + 리스트 표시
- `종각 빨래방` 검색 → 종각 지역 기준 결과 표시 + 지도 이동
- 카테고리 선택 → 해당 카테고리 결과 마커 + 리스트 표시
- 리스트 아이템 탭 → 해당 마커 포커스 + 상세 바텀시트 전환
- 마커 탭 → 상세 바텀시트 직접 오픈

## 3. 바텀시트 모드

바텀시트는 두 가지 독립 컴포넌트로 구성한다.

| 컴포넌트               | 트리거                     | 스냅 포인트 | 내용                  |
| ---------------------- | -------------------------- | ----------- | --------------------- |
| `MapResultBottomSheet` | 검색 실행 / 카테고리 선택  | 30% / 60%   | 결과 리스트           |
| `MapPlaceBottomSheet`  | 리스트 아이템 탭 / 마커 탭 | 32%         | 단일 장소 상세 (기존) |

- 상세 모드 오픈 시 리스트 모드는 닫힘
- 상세 모드 닫힘 시 리스트 모드 다시 열림 (결과가 남아있으면)
- 검색/카테고리 초기화 시 둘 다 닫힘

### 리스트 아이템 표시 정보

- 장소명
- 거리 (현재 지도 중심 기준, `m` / `km`)
- 도로명 주소 (없으면 지번 주소)

## 4. 데이터 흐름

### 검색 실행

```
사용자 검색어 입력 → 검색 버튼 탭
  → RN: SEARCH_KEYWORD { keyword } 전송
  → WebView: ps.keywordSearch(keyword, cb, { location: map.getCenter(), radius: 5000 })
  → 결과 있음:
      마커 표시 + fitBounds
      WebView→RN: SEARCH_RESULT { places: MapPlace[] }
      RN: MapResultBottomSheet 오픈 (리스트 모드)
  → 결과 없음:
      WebView→RN: SEARCH_ZERO_RESULT { keyword }
      RN: 검색창 하단 "검색하신 장소가 존재하지 않습니다" 표시
```

### 카테고리 선택

```
카테고리 칩 탭
  → 기존과 동일하게 SEARCH_CATEGORY 전송
  → WebView 검색 후 결과 있음:
      마커 표시
      WebView→RN: SEARCH_RESULT { places: MapPlace[] }
      RN: MapResultBottomSheet 오픈 (리스트 모드)
```

### 리스트 아이템 탭

```
리스트 아이템 탭
  → RN: FOCUS_MARKER { place } 전송
  → WebView: 해당 마커 인포윈도우 오픈 + 지도 중심 이동
  → MapResultBottomSheet 닫힘
  → MapPlaceBottomSheet 오픈 (상세 모드)
```

### 검색 ↔ 카테고리 상호 배타

- 검색 실행 → 선택된 카테고리 해제, 카테고리 마커 제거, 검색 결과로 교체
- 카테고리 선택 → 검색어 초기화, 검색 마커 제거, 카테고리 결과로 교체
- 마지막에 실행한 쪽이 항상 우선

## 5. postMessage 인터페이스 추가

### RN → WebView

| type             | payload                    | 설명                              |
| ---------------- | -------------------------- | --------------------------------- |
| `SEARCH_KEYWORD` | `{ keyword: string }`      | 키워드 검색 실행                  |
| `FOCUS_MARKER`   | `{ x: string; y: string }` | 특정 마커 포커스 + 지도 중심 이동 |

### WebView → RN

| type                 | payload                  | 설명                                    |
| -------------------- | ------------------------ | --------------------------------------- |
| `SEARCH_RESULT`      | `{ places: MapPlace[] }` | 검색/카테고리 결과 전달 (리스트 표시용) |
| `SEARCH_ZERO_RESULT` | `{ keyword: string }`    | 검색 결과 0건                           |

> 기존 `MARKER_CLICK`은 마커 직접 탭 시 그대로 유지. 리스트 탭은 RN에서 직접 처리.

## 6. 예외 처리

| 상황                       | 처리                                             |
| -------------------------- | ------------------------------------------------ |
| 검색 결과 0건              | 검색창 하단 `검색하신 장소가 존재하지 않습니다`  |
| 검색어 공백/빈 문자열      | 검색 실행하지 않음                               |
| 검색 중 카테고리 선택      | 검색 초기화 + 카테고리 결과로 교체               |
| 카테고리 선택 중 검색 실행 | 카테고리 해제 + 검색 결과로 교체                 |
| 상세 바텀시트 닫기         | 리스트 결과 남아있으면 리스트 바텀시트 다시 표시 |
