# chat 전체 화면 UI 명세

## 목적

홈 화면의 플로팅 버튼으로 진입하는 전체 화면 AI 챗 UI를 정의한다. 사용자는 새 채팅을 시작하거나 이전 채팅을 다시 열고, 청년 정책과 레시피 질문을 입력할 수 있다.

## 범위

### screens/home

- 홈 화면 우하단에 AI 챗 플로팅 버튼을 표시한다.
- 플로팅 버튼은 하단 탭 바와 겹치지 않도록 safe area와 탭 바 높이를 고려해 배치한다.
- 버튼을 누르면 전체 화면 chat UI를 연다.

### features/chat

- 전체 화면 chat UI를 조립한다.
- 새 채팅, 이전 채팅 기록, 닫기, 메시지 목록, 입력창을 제공한다.
- API 응답으로 받은 완성된 AI 답변을 타이핑 애니메이션으로 표시한다.
- AI 답변 content는 `react-native-marked`로 markdown 렌더링한다.

### entities/chat

- 세션/메시지 데이터 조회와 mutation을 제공한다.
- 화면은 entity 내부 파일을 직접 import하지 않고 barrel export를 사용한다.

## 전체 화면 구조

채팅 UI는 화면 전체를 차지한다.

1. 상단 바
2. 기록 영역 또는 기록 패널 진입 버튼
3. 메시지 목록
4. 하단 입력 영역

상단과 하단은 고정 영역으로 두고, 메시지 목록만 세로 스크롤된다.

## 홈 플로팅 버튼

- 위치: 홈 화면 우하단
- 형태: 원형 또는 작은 rounded 버튼
- 내용: AI/채팅을 인지할 수 있는 아이콘 중심
- 접근성 label: `AI 챗 열기`
- 동작: 누르면 chat 전체 화면을 연다.
- 다른 홈 카드나 하단 탭 바의 터치 영역을 가리지 않아야 한다.

## 상단 바

상단 바에는 다음 요소를 배치한다.

- `새 채팅` 액션
- `이전 채팅 기록` 액션
- 닫기 버튼

규칙:

- 닫기 버튼은 우측에 둔다.
- 닫기를 누르면 채팅 화면을 닫고 이전 화면으로 돌아간다.
- 새 채팅을 누르면 서버 세션을 즉시 만들지 않고 빈 대화 상태로 전환한다.
- 빈 대화에서 첫 메시지를 보낼 때 서버 세션을 생성한다.
- 이전 채팅 기록 액션은 세션 목록을 보여준다.
- 상단 바 높이는 고정하고, 긴 텍스트 때문에 높이가 변하지 않아야 한다.

## 이전 채팅 기록 UI

이전 채팅 기록은 전체 화면 안의 목록 영역 또는 slide-in 패널로 제공한다. 구현 시 기존 앱 UI 패턴에 맞는 쪽을 선택한다.

세션 항목 표시:

- 첫 질문 기반 제목
- 마지막 업데이트 시각 또는 상대 시간
- 마지막 메시지 미리보기는 선택 사항
- 이전 채팅 기록 제목 옆에 `최대 5개 저장` 문구를 표시한다.

제목 규칙:

- 제목은 첫 사용자 질문을 사용한다.
- 제목이 길면 한 줄로 표시하고 `...` 처리한다.
- 제목이 비어 있으면 `새 채팅`으로 표시한다.

동작:

- 세션 항목을 누르면 해당 세션 메시지를 불러오고 기록 UI를 닫는다.
- 현재 열린 세션은 선택 상태로 표시할 수 있다.
- 기록이 없으면 빈 상태를 표시한다.
- 첫 메시지를 보내지 않은 빈 대화는 기록 목록에 표시하지 않는다.
- 기록 목록은 서버에서 반환한 최근 5개만 표시한다.
- 삭제 기능은 이번 범위에 포함하지 않는다.

빈 상태:

- 이전 채팅이 없다는 짧은 안내를 보여준다.
- 새 채팅을 시작할 수 있는 액션을 제공한다.

## 메시지 목록

메시지 목록은 사용자 메시지와 AI 메시지를 시간순으로 표시한다.

사용자 메시지:

- 우측 정렬
- 사용자 발화임을 구분할 수 있는 배경색 사용
- 긴 문장은 줄바꿈한다.

AI 메시지:

- 좌측 정렬
- 앱의 기본 배경과 구분되는 말풍선 또는 텍스트 블록으로 표시
- 청년 정책/레시피 안내에 적합한 가독성 있는 줄 간격을 사용
- markdown의 줄바꿈, 목록, 강조 텍스트를 말풍선 안에서 읽기 좋게 표시한다.
- 별도 출처 카드 또는 근거 UI는 만들지 않는다.

스크롤 규칙:

- 새 메시지가 추가되면 목록 하단으로 이동한다.
- 사용자가 과거 메시지를 보고 있는 중이면 강제 스크롤 여부는 구현 시 UX를 확인해 결정한다.
- 키보드가 열려도 마지막 메시지와 입력창이 가려지지 않아야 한다.

빈 대화 상태:

- 새 채팅 또는 메시지가 없는 대화 상태에서는 첫 질문을 유도하는 간단한 빈 상태를 표시한다.
- 예시 질문 버튼은 이번 범위에서 필수는 아니다.

## 입력 영역

입력 영역은 화면 하단에 고정한다.

구성:

- 다중 줄 텍스트 입력
- 전송 버튼

규칙:

- 빈 문자열 또는 공백만 있는 메시지는 전송할 수 없다.
- 전송 중에는 중복 전송을 막는다.
- 전송 중에도 입력창 높이가 과하게 커져 메시지 목록을 밀어내지 않도록 최대 높이를 둔다.
- 키보드가 열리면 입력 영역은 키보드 위에 유지한다.
- 전송 성공 후 입력값을 비운다.
- 전송 실패 시 사용자가 입력했던 메시지를 복구하거나 실패 상태에서 재시도할 수 있어야 한다.

## 응답 생성 상태

메시지를 전송하면 다음 순서로 화면이 변한다.

1. 사용자 메시지를 즉시 목록에 추가한다.
2. AI 응답 영역에 로딩 상태를 표시한다.
3. API 응답을 받으면 로딩 상태를 제거한다.
4. AI 메시지를 빈 content 또는 일부 content로 추가한다.
5. 클라이언트 타이핑 애니메이션으로 content를 점진 표시한다.
6. 타이핑이 끝나면 최종 content를 확정한다.

응답 생성 중에는 새 메시지 전송을 막는 것을 기본으로 한다.

## 클라이언트 타이핑 애니메이션

API는 완성된 답변을 한 번에 반환한다. UI는 이를 스트리밍처럼 보이도록 표시한다.

지연 규칙:

- `.`, `!`, `?` 뒤에는 긴 지연을 준다.
- `,`, 줄바꿈, `:` 뒤에는 중간 지연을 준다.
- 일부 공백과 랜덤 위치에는 짧은 지연을 준다.
- 그 외에는 짧은 기본 지연을 준다.

표시 규칙:

- 한 번에 5~15자 정도의 chunk를 추가 표시한다.
- 타이핑 중인 메시지는 `isTyping` 상태로 구분한다.
- 사용자가 화면을 닫거나 다른 세션으로 이동하면 진행 중인 타이머를 정리한다.
- 타이핑 완료 후 서버에서 받은 최종 AI 메시지 content와 동일해야 한다.

저장 규칙:

- 서버 저장은 API 응답 기준으로 완료된 상태다.
- 타이핑 애니메이션은 모바일 화면 표시용 상태만 갱신한다.
- 세션 기록을 다시 열면 저장된 완성 메시지를 즉시 표시한다.

## 에러 상태

세션 목록 조회 실패:

- 기록 영역에 에러 상태를 표시한다.
- 현재 채팅 화면 전체를 막지 않는다.

메시지 조회 실패:

- 메시지 목록 영역에 에러 상태를 표시한다.
- 재시도 액션을 제공한다.

메시지 전송 실패:

- 사용자 메시지와 AI 로딩 말풍선은 제거한다.
- 입력창에는 사용자가 작성했던 메시지를 복구한다.
- AI 로딩 상태는 제거한다.
- Toast로 `답변을 생성하지 못했어요. 잠시 후 다시 시도해주세요.`를 표시한다.
- 실패한 질문은 이전 채팅 기록에 남지 않아야 한다.

AI 응답 실패:

- API가 502를 반환하면 AI 응답 생성 실패 안내를 표시한다.
- 사용자가 같은 메시지를 다시 시도할 수 있어야 한다.

## 로딩 상태

- 세션 목록 로딩은 기록 영역 안에서만 표시한다.
- 세션 메시지 로딩은 메시지 목록 영역에 skeleton 또는 간단한 loading indicator로 표시한다.
- 메시지 전송 loading은 AI 말풍선 위치에 표시한다.
- 전체 화면을 불필요하게 막지 않는다.

## 접근성

- 닫기, 새 채팅, 이전 채팅 기록, 전송 버튼에는 접근성 label을 둔다.
- 전송 버튼 disabled 상태가 시각적으로 구분되어야 한다.
- 메시지 말풍선 텍스트는 작은 화면에서도 읽을 수 있는 크기를 유지한다.
- 텍스트가 버튼이나 말풍선 밖으로 넘치지 않아야 한다.

## 제외 범위

- 실제 SSE/chunked streaming
- 채팅 기록 삭제
- 별도 출처 카드 또는 근거 UI
- 비로그인 채팅
- 모바일 앱의 Google API 직접 호출

## 검증 기준

- 홈 플로팅 버튼을 누르면 전체 화면 채팅 UI가 열린다.
- 닫기 버튼을 누르면 이전 화면으로 돌아간다.
- 새 채팅을 누르면 서버 세션을 만들지 않고 빈 메시지 목록으로 전환된다.
- 빈 메시지 목록에서 첫 메시지를 전송하면 서버 세션이 생성되고 기록으로 인정된다.
- 이전 채팅 기록에서 세션을 선택하면 해당 메시지 목록이 표시된다.
- 이전 채팅 기록 제목 옆에 최대 5개 저장 가능 문구가 표시된다.
- 이전 채팅 기록은 최근 5개를 초과하지 않는다.
- 첫 메시지를 보내지 않은 빈 대화는 이전 채팅 기록에 표시되지 않는다.
- 긴 첫 질문 제목은 기록 목록에서 `...`으로 생략된다.
- 메시지 전송 후 사용자 메시지가 즉시 표시된다.
- API 응답 완료 후 AI 메시지가 타이핑 애니메이션으로 점진 표시된다.
- 타이핑 완료 후 AI 메시지 content는 API 응답 content와 일치한다.
- AI 메시지는 markdown 형식으로 렌더링된다.
- 메시지 전송 실패 시 Toast가 표시되고 실패한 메시지는 기록에 남지 않는다.
- 전송 중 중복 전송이 발생하지 않는다.
- 작은 화면에서 상단 바, 메시지 목록, 입력 영역이 서로 겹치지 않는다.
