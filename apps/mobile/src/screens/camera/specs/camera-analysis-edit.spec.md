# camera 분석 결과 수정 화면 명세

## 목적

카메라 분석 직후 AI가 추출한 영수증/식재료 데이터를 사용자가 확인하고 수정한 뒤 저장할 수 있는 UI를 제공한다.

이번 범위는 UI 구현과 실제 저장 API 연동을 포함한다.

## 범위

### screens/camera

- 분석 직후 표시되는 결과 확인/수정 화면을 조립한다.
- 공통 Header, 안내 카드, 구매일 영역, 항목 리스트, 하단 고정 액션을 배치한다.
- `source` 값에 따라 영수증/식재료 화면 구성을 분기한다.

### entities/camera

- 분석 결과 항목을 수정할 수 있는 폼 UI를 제공한다.
- 항목 추가/삭제와 기본 입력 상태를 관리할 수 있는 UI 구조를 제공한다.

## 진입 조건

- 촬영/분석이 성공하면 결과 확인/수정 화면을 표시한다.
- 화면은 `useCameraStore.analysisResult` 또는 UI 확인용 목 데이터를 기반으로 렌더링한다.
- `source=RECEIPT`이면 영수증 분석 결과 화면으로 동작한다.
- `source=INGREDIENT`이면 식재료 분석 결과 화면으로 동작한다.

## 화면 구성

- 공통 Header
  - 제목: `AI 분석 결과`
- 안내 카드
  - 안내 문구: `AI 분석 결과입니다. 정확하지 않은 항목은 수정/삭제하고 누락된 항목은 추가해 주세요.`
  - 분석 출처: `영수증 촬영` 또는 `식재료 촬영`
- 영수증 전용 영역
  - 구매일 입력/표시
  - 저장 위치 선택 영역
    - 기본 선택: `장보기 내역` + `My 냉장고`
    - 선택지: `장보기 내역`, `My 냉장고`
    - 최소 하나 이상 선택
  - 항목별 가격 입력
  - 예상 총 금액 표시
- 식재료 전용 규칙
  - 저장 위치 선택 영역을 표시하지 않는다.
  - 저장 대상은 항상 `My 냉장고`로 간주한다.
  - 구매일, 가격, 총액을 표시하지 않는다.
- 항목 리스트
  - 총 항목 수
  - `항목 추가` 버튼
  - 품목별 카드
    - 삭제 버튼
    - 품목명
    - 카테고리
    - 수량
    - 단위
    - 영수증일 때만 가격
- 하단 고정 액션
  - `취소`
  - `저장하기`
  - 영수증이면 저장 위치와 총액을 표시한다.
  - 식재료이면 저장 위치 선택과 총액 없이 저장 액션 중심으로 표시한다.

## 저장 API 연동

- `저장하기`를 누르면 폼 검증을 먼저 수행한다.
- 폼 검증을 통과한 경우 `POST /api/ai/camera/results`를 호출한다.
- 요청 본문은 API 명세 `apps/api/src/services/camera/specs/camera-result-save-api.spec.md`를 따른다.
- 요청 본문에서 필요 없는 optional 필드는 생략하고 `null`은 보내지 않는다.
- 저장 위치 매핑:
  - `장보기 내역`은 `PURCHASE`로 보낸다.
  - `My 냉장고`는 `FRIDGE`로 보낸다.
- 영수증에서 저장 위치가 `장보기 내역`을 포함하면 `purchaseDate`를 `YYYY-MM-DD` 형식으로 보낸다.
- 영수증에서 저장 위치가 `장보기 내역`을 포함하면 각 item의 `price`를 보낸다.
- 저장 위치가 `My 냉장고`를 포함하면 각 item의 `category`를 보낸다.
- 식재료 결과는 `destinations: ["FRIDGE"]`로 보낸다.
- 저장 요청 중에는 저장 버튼을 비활성화하고 중복 요청을 막는다.
- 저장 성공 시 공통 Toast로 `AI 분석 결과를 저장했어요.`를 표시한다.
- 저장 성공 시 `useCameraStore.clearAnalysisResult()`로 분석 결과를 비운다.
- 저장 성공 시 홈 화면으로 `replace` 이동한다.
- 저장 실패 시 현재 화면에 머무르고 공통 Toast로 `저장하지 못했어요. 잠시 후 다시 시도해주세요.`를 표시한다.

## 디자인 적용 기준

- Figma는 전체 구조와 배치만 참고한다.
- 색상, 보더, 배경, 텍스트 톤은 프로젝트 디자인 토큰을 우선한다.
  - `main-100`, `main-10`
  - `surface-card`, `surface-default`
  - `gray-*`
  - `point-100`
- 공통 `Header`를 사용한다.
- 입력/버튼은 현재 프로젝트의 간격, 라운드, 타이포그래피에 맞춰 구현한다.

## 상태

| 상태           | 요구사항                                              |
| -------------- | ----------------------------------------------------- |
| 분석 결과 있음 | 분석 결과를 폼 초기값으로 사용한다.                   |
| 분석 결과 없음 | UI 확인용 목 데이터를 사용해 화면을 렌더링할 수 있다. |
| 영수증         | 구매일, 가격, 총액, 저장 위치 선택을 표시한다.        |
| 식재료         | 구매일, 가격, 총액, 저장 위치 선택을 숨긴다.          |
| 항목 없음      | 항목 추가 버튼으로 새 항목을 만들 수 있다.            |
| 저장 중        | 저장 버튼을 비활성화하고 중복 저장을 막는다.          |
| 저장 성공      | Toast 표시, 분석 결과 초기화, 홈 이동을 수행한다.     |
| 저장 실패      | 현재 화면에 머무르고 Toast로 오류를 표시한다.         |

## 오류 검사

- `저장하기`를 누르면 폼 값을 검증한다.
- 오류가 있으면 저장을 진행하지 않고 각 입력 아래에 오류 메시지를 표시한다.
- 품목명, 수량, 단위는 모든 source에서 필수다.
- 카테고리는 `My 냉장고`에 저장되는 경우 필수다.
- 영수증 분석 결과에서 저장 위치가 `장보기 내역`만 선택된 경우:
  - 카테고리는 필수가 아니다.
- 수량은 기존 `cameraAnalysisItemSchema` 규칙을 따른다.
- 영수증 분석 결과에서 저장 위치가 `My 냉장고`만 선택된 경우:
  - 구매일은 필수가 아니다.
  - 가격은 필수가 아니다.
- 영수증 분석 결과에서 저장 위치가 `장보기 내역`을 포함하는 경우:
  - 구매일은 필수다.
  - 각 항목의 가격은 필수다.
  - 가격 숫자 규칙은 기존 `cameraAnalysisItemSchema` 규칙을 따른다.
- 저장 위치는 최소 하나 이상 선택해야 한다.

## 검증 기준

- `RECEIPT` 결과에서는 구매일, 가격, 총액, 저장 위치 선택이 보인다.
- `RECEIPT`의 저장 위치 기본값은 `장보기 내역` + `My 냉장고`다.
- `INGREDIENT` 결과에서는 저장 위치 선택, 구매일, 가격, 총액이 보이지 않는다.
- `INGREDIENT` 결과는 저장 대상이 내부적으로 `My 냉장고`로 처리된다.
- 항목 추가/삭제와 기본 텍스트 수정 UI가 동작한다.
- 필수 입력값이 비어 있거나 숫자 규칙을 통과하지 못하면 필드별 오류 메시지가 표시되고 저장이 막힌다.
- 영수증에서 `My 냉장고`만 선택하면 구매일/가격 빈 값으로 저장 시도해도 오류가 표시되지 않는다.
- 영수증에서 `장보기 내역`을 포함하면 구매일/가격 빈 값으로 저장 시도 시 오류가 표시된다.
- 영수증에서 `장보기 내역`만 선택하면 카테고리 오류가 표시되지 않고, 예상 총 금액이 표시된다.
- 영수증에서 `장보기 내역` + `My 냉장고`를 선택하고 저장하면 `PURCHASE`, `FRIDGE`를 모두 포함해 API를 호출한다.
- 영수증에서 `장보기 내역`만 선택하고 저장하면 item `category`를 보내지 않아도 API 요청을 만들 수 있다.
- 영수증에서 `My 냉장고`만 선택하고 저장하면 `purchaseDate`와 item `price`를 보내지 않아도 API 요청을 만들 수 있다.
- 식재료에서 저장하면 `destinations`는 `["FRIDGE"]`로 API 요청을 만든다.
- 저장 성공 시 Toast가 표시되고, 분석 결과 store를 비운 뒤 홈으로 이동한다.
- 저장 실패 시 현재 화면에 머무르고 Toast로 오류가 표시된다.
- 프로젝트 색상/헤더/간격 톤과 크게 어긋나지 않는다.
