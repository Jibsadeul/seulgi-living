# camera 분석 테스트 화면 명세

## 목적

camera 화면에서 촬영 이미지를 분석하고, 응답이 요구된 JSON 형식으로 내려오는지 페이지에서 바로 확인한다.

## 범위

### screens/camera

- 라우트에서 전달받은 `mode`, `imageUri`를 사용해 테스트 화면을 조립한다.
- 이미지 미리보기, 이미지 교체 버튼, 분석 실행 버튼, 로딩/에러/결과 영역을 배치한다.
- 결과는 우선 편집 UI 없이 JSON 그대로 출력한다.

## 진입 조건

- `features/camera-capture`가 `/(stack)/camera`로 이동할 때 `mode`와 `imageUri`를 전달한다.
- `mode=receipt`면 API 요청의 `source`는 `RECEIPT`다.
- `mode=ingredient`면 API 요청의 `source`는 `INGREDIENT`다.

## 화면 구성

- 상단 제목
  - `receipt`: `영수증 촬영`
  - `ingredient`: `식재료 촬영`
- 촬영 이미지 미리보기
- `재촬영` 버튼
- `앨범에서 선택` 버튼
- `분석 시작` 버튼
- 분석 중 로딩 영역
- 실패 메시지 영역
- 성공 시 JSON 문자열 출력 영역

## 상태

| 상태        | 요구사항                                                       |
| ----------- | -------------------------------------------------------------- |
| 초기        | 이미지 미리보기와 이미지 교체 버튼, 분석 시작 버튼을 표시한다. |
| 분석 중     | 버튼을 비활성화하고 중복 요청을 막는다.                        |
| 성공        | 응답 JSON을 페이지에 그대로 표시한다.                          |
| 실패        | 오류 메시지를 표시하고 다시 시도할 수 있다.                    |
| 이미지 없음 | 분석 버튼을 숨기고 안내 문구를 표시한다.                       |

## 이미지 교체

- 촬영 이미지 미리보기 아래에 `재촬영`과 `앨범에서 선택` 버튼을 표시한다.
- 두 버튼은 분석 시작 버튼보다 낮은 위계의 보조 버튼으로 표시한다.
- `재촬영` 또는 `앨범에서 선택`으로 새 이미지를 선택하면 기존 분석 결과와 에러 메시지를 비운다.
- 이미지 교체 후 분석을 실행하면 새 이미지 URI를 기준으로 base64와 MIME 타입을 만든다.

## 검증 기준

- `receipt` 촬영 후 결과에 `source=RECEIPT`가 표시된다.
- `receipt` 촬영 후 결과에 `date`가 ISO 8601 timestamp 문자열 또는 `null`로 표시된다.
- `ingredient` 촬영 후 결과에 `source=INGREDIENT`가 표시된다.
- `ingredient` 촬영 후 결과에 `date=null`이 표시된다.
- `ingredient` 결과 item의 `price`가 모두 `null`로 보인다.
- 분석 실패 시 오류 메시지가 페이지에 표시된다.
