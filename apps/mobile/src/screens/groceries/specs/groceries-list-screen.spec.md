# 이번 달 장보기 내역 화면 명세

## 목적

사용자가 월별 구매일별 장보기 내역을 확인한다.

이번 작업은 장보기 내역 조회 화면만 구현한다. 예산 관리, 항목 추가, 수정, 삭제 기능은 후속 작업에서 별도 구현한다.

## 화면 진입

- 관련 진입점에서 장보기 내역 화면으로 이동한다.
- 화면 최초 진입 시 현재 날짜 기준 `year`, `month`로 조회한다.

## 데이터 흐름

### 월 선택

- 화면 상단에 `YYYY년 M월`을 표시한다.
- 좌측 아이콘 버튼을 누르면 이전 달로 이동한다.
- 우측 아이콘 버튼을 누르면 다음 달로 이동한다.
- 월 이동 버튼은 배경 없는 아이콘 버튼으로 표시한다.
- 월 변경 시 아래 API를 같은 `year`, `month`로 다시 조회한다.

### API

- 장보기 내역 목록: `GET /api/groceries?year={year}&month={month}`

모바일 API 응답 타입은 `@repo/contract`의 groceries Zod 스키마를 기준으로 검증한다.

## 표시 정보

### 헤더

- 제목: `이번 달 장보기 내역`
- 기존 앱 헤더/상단바 스타일과 맞춘다.

### 장보기 내역 목록

- API 응답의 날짜 그룹 단위로 카드 또는 섹션을 표시한다.
- 각 날짜 그룹에는 다음을 표시한다.
  - 날짜: `YYYY. MM. DD (요일)`
  - 일별 합계: 원화 형식
  - 항목 목록
- 각 항목에는 다음을 표시한다.
  - 이름
  - 수량 텍스트: `quantityText`가 있으면 표시, 없으면 생략
  - 가격: 원화 형식
  - 옵션 버튼: 점 3개 아이콘. 수정/삭제 옵션 진입점으로 사용한다.

## UI 상태

### 로딩

- 목록 영역에 기존 앱 톤에 맞는 로딩 상태를 표시한다.

### 에러

- 조회 실패 시 화면 내 에러 상태를 표시한다.
- 다시 시도 버튼을 제공한다.
- 다시 시도 시 list 쿼리를 재요청한다.

### 빈 상태

- 해당 월 장보기 내역이 없으면 빈 상태 문구를 표시한다.
- 예: `이번 달 장보기 내역이 없어요.`
- 추가 기능은 후속 작업이므로, 추가 버튼은 비활성화하지 않고 진입점만 유지할 수 있다.

## 디자인 기준

- Figma URL의 레이아웃 구조만 참고한다.
- 색상은 프로젝트 디자인 토큰을 사용한다.
  - 배경: `bg-surface-card`
  - 카드: `bg-surface-default`
  - 주요 강조: `main-100`
  - 텍스트: `gray-90`, `gray-70`, `gray-50`
  - 보더: `gray-20`
- 기존 홈/정책/레시피 화면의 카드, 간격, 타이포그래피 스타일과 맞춘다.
- Figma의 갈색 계열 색상은 그대로 복사하지 않고 프로젝트 토큰으로 치환한다.

## 이번 작업 제외 범위

- 장보기 항목 추가 API 연동
- 장보기 항목 수정/삭제 실제 동작
- 예산 요약 조회
- 예산 관리 카드 UI
- 예산 수정/저장
- 영수증 촬영 결과에서 장보기 화면으로 이동하는 상세 플로우

## 검증 기준

- 현재 월 기준 list API를 호출한다.
- 월 이동 시 변경된 `year/month`로 재조회한다.
- 목록이 날짜별 그룹으로 표시된다.
- 빈 목록이면 빈 상태가 표시된다.
- API 실패 시 에러 상태와 다시 시도 동작이 제공된다.
