# 이번 달 장보기 내역 화면 명세

## 목적

사용자가 월별 구매일별 장보기 내역을 확인하고, 직접 입력으로 장보기 내역을 추가한다.

이번 작업은 장보기 내역 조회 화면과 직접 입력 추가 기능을 구현한다. 예산 관리, 항목 수정, 삭제 기능은 후속 작업에서 별도 구현한다.

## 화면 진입

- 관련 진입점에서 장보기 내역 화면으로 이동한다.
- 화면 최초 진입 시 현재 날짜 기준 `year`, `month`로 조회한다.

## 데이터 흐름

### 월 선택

- 화면 상단에 `YYYY년 M월`을 표시한다.
- 좌측 아이콘 버튼을 누르면 이전 달로 이동한다.
- 우측 아이콘 버튼을 누르면 다음 달로 이동한다.
- 월 이동 버튼은 배경 없는 아이콘 버튼으로 표시한다.
- 월 변경 시 아래 API를 같은 `year`, `month`로 다시 조회한다.

### API

- 장보기 내역 목록: `GET /api/groceries?year={year}&month={month}`
- 예산 및 지출 요약: `GET /api/groceries/summary?year={year}&month={month}`
- 예산 설정: `PUT /api/groceries/budget?year={year}&month={month}`
- 장보기 내역 직접 추가: `POST /api/groceries`

모바일 API 응답 타입은 `@repo/contract`의 groceries Zod 스키마를 기준으로 검증한다.

직접 추가 요청 바디는 `@repo/contract`의 `createGroceryBodySchema`를 기준으로 한다.

예산 설정 요청 바디는 `@repo/contract`의 `putGroceryBudgetBodySchema`를 기준으로 한다.

## 표시 정보

### 헤더

- 제목: `이번 달 장보기 내역`
- 기존 앱 헤더/상단바 스타일과 맞춘다.

### 장보기 내역 목록

- 연/월 선택 영역 아래, 장보기 내역 목록 위에 `GroceryBudgetSummaryCard`를 표시한다.
- 카드는 선택된 `year`, `month` 기준 summary API 응답을 사용한다.
- 카드는 요약만 표시하고 예산 수정 입력은 카드 내부에 두지 않는다.
- 카드의 primary action은 다음 값을 사용한다.
  - label: `예산 수정`
  - iconName: `create-outline`
- 장보기 내역 화면의 예산 요약 카드는 `더 보기` 버튼을 표시하지 않는다.
- 카드의 primary action은 예산 수정 바텀시트를 연다.
- API 응답의 날짜 그룹 단위로 카드 또는 섹션을 표시한다.
- 각 날짜 그룹에는 다음을 표시한다.
  - 날짜: `YYYY. MM. DD (요일)`
  - 일별 합계: 원화 형식
  - 항목 목록
- 각 항목에는 다음을 표시한다.
  - 이름
  - 수량 텍스트: `quantityText`가 있으면 표시, 없으면 생략
  - 가격: 원화 형식
  - 옵션 버튼: 점 3개 아이콘. 수정/삭제 옵션 진입점으로 사용한다.

### 플로팅 추가 버튼

- 화면 우하단에 장보기 추가 FAB를 표시한다.
- FAB를 누르면 backdrop과 함께 두 가지 액션 옵션을 표시한다.
  - `영수증 등록`
  - `직접 입력`
- backdrop 또는 닫기 버튼을 누르면 액션 메뉴를 닫는다.
- `영수증 등록`을 누르면 소스 선택 단계로 전환한다.
  - 이전 버튼
  - `촬영`
  - `앨범 선택`
- 소스 선택 단계에서 메인 FAB는 이전 버튼으로 바뀐다.
- 소스 선택 단계의 이전 버튼을 누르면 `영수증 등록`, `직접 입력` 옵션 단계로 돌아간다.
- `촬영` 또는 `앨범 선택`으로 이미지를 선택하면 `mode=receipt`, `imageUri`를 camera route에 전달한다.
- `직접 입력`을 누르면 FAB 메뉴를 닫고 직접 입력 바텀시트를 연다.

### 직접 입력 바텀시트

- `직접 입력` 바텀시트는 기존 앱의 바텀시트 스타일과 맞춘다.
- 입력 필드는 다음을 포함한다.
  - 품목명: 필수, 1자 이상 50자 이하
  - 가격: 필수, 0 이상 정수
  - 구매일: 필수, `YYYY-MM-DD` 형식. 기본값은 선택된 월의 오늘 날짜이며, 선택된 월이 현재 월이 아니면 선택된 월의 1일이다.
  - 수량: 선택, 1자 이상 20자 이하. 비어 있으면 요청에서 생략한다.
- 저장 버튼은 필수값이 유효할 때만 활성화한다.
- 저장 중에는 저장 버튼을 비활성화하고 중복 제출을 막는다.
- 취소, 닫기, 바텀시트 pan down으로 닫을 수 있다.
- 저장 성공 시 바텀시트를 닫고 장보기 목록을 다시 조회한다.
- 저장 실패 시 바텀시트를 유지하고 실패 토스트를 표시한다.
- 입력 영역은 필요 시 스크롤되며, 하단 취소/저장 버튼은 시트 내부 하단 액션 영역에서 safe area를 반영해 가려지지 않아야 한다.
- safe area가 없는 web 환경에서도 하단 액션 버튼 아래에 충분한 여백을 둔다.

### 예산 수정 바텀시트

- `예산 수정` 바텀시트는 기존 앱의 바텀시트 스타일과 맞춘다.
- 바텀시트는 다음 정보를 표시한다.
  - 제목: `이번 달 예산 수정`
  - 현재 예산: 현재 summary의 `budget` 값. 설정되지 않았으면 `미설정`
  - 새 예산 입력 필드
  - 하단 액션: `저장하기`
- 새 예산 입력은 숫자만 허용한다.
- `0`은 유효한 예산으로 허용한다.
- 최대 예산은 `99,999,999`이다.
- 빈 값은 저장할 수 없다.
- 저장 중에는 저장 버튼을 비활성화하고 로딩 텍스트를 표시한다.
- 저장 성공 시 바텀시트를 닫고 선택된 월의 summary query를 다시 조회한다.
- 저장 실패 시 바텀시트를 유지하고 실패 토스트 또는 시트 내부 에러 텍스트를 표시한다.

## UI 상태

### 로딩

- 목록 영역에 기존 앱 톤에 맞는 로딩 상태를 표시한다.

### 에러

- 조회 실패 시 화면 내 에러 상태를 표시한다.
- 다시 시도 버튼을 제공한다.
- 다시 시도 시 list 쿼리를 재요청한다.

### 빈 상태

- 해당 월 장보기 내역이 없으면 빈 상태 문구를 표시한다.
- 예: `이번 달 장보기 내역이 없습니다.`
- 추가 버튼은 빈 상태에서도 동일하게 동작한다.

## 디자인 기준

- Figma URL의 레이아웃 구조만 참고한다.
- 색상은 프로젝트 디자인 토큰을 사용한다.
  - 배경: `bg-surface-card`
  - 카드: `bg-surface-default`
  - 주요 강조: `main-100`
  - 텍스트: `gray-90`, `gray-70`, `gray-50`
  - 보더: `gray-20`
- 기존 홈/정책/레시피 화면의 카드, 간격, 타이포그래피 스타일과 맞춘다.
- Figma의 갈색 계열 색상은 그대로 복사하지 않고 프로젝트 토큰으로 치환한다.

## 이번 작업 제외 범위

- 장보기 항목 수정/삭제 실제 동작
- 영수증 촬영 결과에서 장보기 화면으로 이동하는 상세 플로우

## 검증 기준

- 현재 월 기준 list API를 호출한다.
- 월 이동 시 변경된 `year/month`로 재조회한다.
- 월 이동 시 변경된 `year/month`로 summary API를 조회한다.
- 연/월 선택 아래에 예산 요약 카드가 표시된다.
- 예산 수정 액션을 누르면 예산 수정 바텀시트가 열린다.
- 유효한 예산을 저장하면 `PUT /api/groceries/budget`이 호출된다.
- 저장 성공 시 바텀시트가 닫히고 summary query가 다시 조회된다.
- 목록이 날짜별 그룹으로 표시된다.
- 빈 목록이면 빈 상태가 표시된다.
- API 실패 시 에러 상태와 다시 시도 동작이 제공된다.
- `직접 입력`을 누르면 바텀시트가 열린다.
- 유효한 직접 입력 값을 저장하면 `POST /api/groceries`가 호출된다.
- 저장 성공 시 바텀시트가 닫히고 목록이 다시 조회된다.
