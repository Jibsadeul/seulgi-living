# 장보기 항목 수정 명세

## 목적

기존 장보기 항목의 이름, 금액, 구매일, 수량을 수정한다.

## 데이터 흐름

- 드롭다운 "수정" 탭 → 해당 항목 데이터가 채워진 입력 바텀시트 열림
- 저장 버튼 탭 → `PUT /api/groceries/:id` 호출
- 성공 → 바텀시트 자동 닫기 + 리스트 자동 갱신 + 성공 토스트
- 실패 → 에러 토스트 (바텀시트 유지)

## UI

- `GroceryDirectInputSheet`를 수정 모드로 재사용
- 헤더 타이틀: "추가" → "수정"
- 저장 버튼 레이블: "추가" → "수정"
- 폼 초기값: 선택된 항목의 기존 데이터로 채움
- 나머지 동작(유효성 검사, 달력, 입력 필드)은 추가와 동일

## 특이 사항

- 저장 성공 후 자동으로 바텀시트 닫기
- 수정 중 로딩 상태는 저장 버튼 비활성화로 처리

## 제외 범위

- 삭제 확인 다이얼로그
