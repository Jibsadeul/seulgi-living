# Home 화면 명세

## 목적

홈 화면은 사용자가 앱을 열었을 때 가장 먼저 보는 대시보드 화면이다. 가계부 현황, 냉장고 임박 재료, 즐겨찾기한 레시피, 즐겨찾기한 청년정책을 한 화면에서 빠르게 확인하고 각 상세 기능으로 이동할 수 있어야 한다.

참고 디자인: `local/design/home.png`

## 범위

### screens/home

Home 화면은 여러 도메인의 데이터를 조합해 보여주는 화면 조립 계층이다.

- 상단 인사 영역과 설정 진입 버튼을 표시한다.
- 가계부 요약 카드를 표시한다.
- 냉장고 미리보기 섹션을 표시한다.
- 레시피 즐겨찾기 섹션을 표시한다.
- 청년정책 즐겨찾기 섹션을 표시한다.
- 하단 탭 바에서 Home 탭이 선택된 상태로 보여야 한다.
- 데이터 조회, 저장, 정렬, 필터링 같은 도메인 로직은 직접 구현하지 않고 feature/entity로 위임한다.

### features

Home 화면에서 필요한 사용자 행동 단위다. 이 화면 spec에서는 연결 지점만 정의한다.

| feature           | 요구사항                                              | Home 화면에서의 진입점                |
| ----------------- | ----------------------------------------------------- | ------------------------------------- |
| 가계부 소비 기록  | 사용자가 이번 달 지출 내역을 추가한다.                | 가계부 요약 카드의 `+ 소비 기록` 버튼 |
| 가계부 지출 통계  | 사용자가 이번 달 지출 통계 화면으로 이동한다.         | 가계부 요약 카드의 `지출 통계` 버튼   |
| 냉장고 목록/상세  | 사용자가 냉장고 재료 목록 또는 재료 상세를 확인한다.  | 냉장고 섹션 `더 보기`, 재료 카드      |
| 레시피 즐겨찾기   | 사용자가 즐겨찾기한 레시피 목록 또는 상세를 확인한다. | 레시피 섹션 `더 보기`, 레시피 카드    |
| 청년정책 즐겨찾기 | 사용자가 즐겨찾기한 정책 목록 또는 상세를 확인한다.   | 정책 섹션 `더 보기`, 정책 카드        |
| 설정              | 사용자가 앱 설정 화면으로 이동한다.                   | 상단 설정 아이콘                      |

### entities

Home 화면은 아래 entity 데이터를 사용한다. API DTO는 `@repo/contract`의 스키마를 기준으로 하고, 화면 전용 mock 데이터는 실제 연동 전까지만 허용한다.

| entity    | 필요한 표시 데이터                                                          |
| --------- | --------------------------------------------------------------------------- |
| members   | 사용자 닉네임                                                               |
| groceries | 이번 달 사용금액, 이번 달 예산, 지출 진행률                                 |
| fridge    | 재료 ID, 재료명, 이미지, 남은 소비기한 일수                                 |
| recipes   | 레시피 ID, 레시피명, 대표 이미지                                            |
| policies  | 정책 ID, 정책명, 설명, 마감까지 남은 일수, 마감 라벨, 아이콘 또는 분류 표시 |

## 화면 구성

### 1. 상단 헤더

- 배경색은 메인 오렌지 계열을 사용한다.
- 안전 영역을 반영해 상태바 아래부터 콘텐츠가 시작되어야 한다.
- 인사 문구 `반가워요!`를 표시한다.
- 사용자명은 `{nickname}님` 형식으로 표시한다.
- 사용자명이 없으면 서비스 기본명 또는 게스트용 문구를 표시한다.
- 우측에는 설정 아이콘 버튼을 표시한다.

### 2. 가계부 요약 카드

- 헤더 내부에 흰색 카드로 표시한다.
- 카드 제목은 `가계부 요약`이다.
- 우측에 `더 보기 >` 액션을 표시한다.
- `이번 달 사용금액` 라벨과 실제 사용금액을 원화 형식으로 표시한다.
- 우측에는 `예산 {amount}원`을 표시한다.
- 진행 바는 `이번 달 사용금액 / 이번 달 예산` 비율을 표시한다.
- 하단에는 `+ 소비 기록`, `지출 통계` 버튼을 나란히 배치한다.
- 예산이 0원이거나 없으면 진행률 계산에서 0으로 처리하고 레이아웃이 깨지지 않아야 한다.

### 3. 냉장고 미리보기

- 섹션 제목은 `냉장고 미리보기`다.
- 우측에 `더 보기 >` 액션을 표시한다.
- 가로 스크롤 카드 목록으로 표시한다.
- 각 카드는 재료 이미지, D-day 배지, 재료명을 포함한다.
- 디자인 기준 노출 예시는 `우유 900ml`, `양상추`, `부채살`이다.
- 이미지가 없으면 정해진 placeholder를 표시한다.
- 재료명은 한 줄 기준으로 표시하고 길면 말줄임 처리한다.

### 4. 레시피 즐겨찾기

- 섹션 제목은 `레시피 즐겨찾기`다.
- 우측에 `더 보기 >` 액션을 표시한다.
- 가로 스크롤 카드 목록으로 표시한다.
- 각 카드는 대표 이미지와 레시피명을 포함한다.
- 디자인 기준 노출 예시는 `영양만점 비빔밥`, `폭신폭신 계란말이`, `얼큰한 김치찌개`다.
- 레시피명은 최대 2줄까지 표시한다.

### 5. 청년정책 즐겨찾기

- 섹션 제목은 `청년정책 즐겨찾기`다.
- 우측에 `더 보기 >` 액션을 표시한다.
- 세로 카드 목록으로 표시한다.
- 각 카드는 정책 아이콘 또는 분류 표시, 정책명, 설명, D-day 배지, 마감 라벨을 포함한다.
- 디자인 기준 노출 예시는 `OO시 청년 월세 지원`, `자취생 전기세 감면 정책`이다.
- 정책명과 설명은 길면 말줄임 처리한다.

### 6. 하단 탭 바

- Home, 요리, 중앙 카메라, 정책, Map 탭을 표시한다.
- Home 탭이 선택된 상태여야 한다.
- 중앙 카메라 탭은 메인 오렌지 원형 버튼으로 강조한다.
- 이 spec은 탭 바 자체의 구현보다 Home 화면에서 보이는 선택 상태를 검증 대상으로 둔다.

## D-day 배지 규칙

| 조건                 | 표시                      | 색상 의도                             |
| -------------------- | ------------------------- | ------------------------------------- |
| `daysLeft <= 1`      | `D-1` 또는 해당 일수      | 긴급, 붉은 계열                       |
| `2 <= daysLeft <= 3` | `D-{daysLeft}`            | 임박, 주황 계열                       |
| `4 <= daysLeft <= 7` | `D-{daysLeft}`            | 주의, 노랑 또는 초록 계열             |
| `daysLeft > 7`       | `D-{daysLeft}`            | 기본, 회색 계열                       |
| `daysLeft === 0`     | `D-Day`                   | 당일 마감                             |
| `daysLeft < 0`       | 표시하지 않거나 만료 상태 | 만료 데이터는 기본 목록에서 제외 권장 |

정확한 색상 토큰은 `shared/ui/DDayBadge` 또는 디자인 토큰을 따른다.

## 데이터 상태

### loading

- 화면 전체를 빈 화면으로 두지 않는다.
- 각 섹션별 skeleton 또는 고정 높이 placeholder를 사용해 레이아웃 점프를 줄인다.

### error

- 특정 섹션 데이터 조회 실패는 해당 섹션 안에서만 에러 상태를 표시한다.
- Home 전체 진입을 막지 않는다.
- 재시도 액션은 실제 API 연동 시 feature/entity 쿼리 정책에 맞춘다.

### empty

| 섹션              | 빈 상태 문구 기준                                                |
| ----------------- | ---------------------------------------------------------------- |
| 냉장고 미리보기   | 등록된 냉장고 재료가 없음을 안내하고 재료 추가 진입을 제공한다.  |
| 레시피 즐겨찾기   | 즐겨찾기한 레시피가 없음을 안내하고 레시피 탐색 진입을 제공한다. |
| 청년정책 즐겨찾기 | 즐겨찾기한 정책이 없음을 안내하고 정책 탐색 진입을 제공한다.     |

## 상호작용

- 설정 아이콘을 누르면 설정 화면으로 이동한다.
- 가계부 요약의 `더 보기 >`를 누르면 가계부 상세 또는 월간 요약 화면으로 이동한다.
- `+ 소비 기록`을 누르면 소비 기록 추가 플로우로 이동한다.
- `지출 통계`를 누르면 지출 통계 화면으로 이동한다.
- 각 섹션의 `더 보기 >`를 누르면 해당 도메인의 전체 목록으로 이동한다.
- 냉장고 재료 카드를 누르면 재료 상세로 이동한다.
- 레시피 카드를 누르면 레시피 상세로 이동한다.
- 정책 카드를 누르면 정책 상세로 이동한다.

라우트명이 확정되지 않은 경우 구현 시점에 Expo Router 구조를 확인해 연결한다.

## 검증 기준

- `local/design/home.png`의 주요 정보 구조와 섹션 순서가 일치한다.
- 작은 화면에서도 카드 텍스트와 버튼 텍스트가 겹치지 않는다.
- 모든 가로 스크롤 섹션은 화면 밖 항목이 자연스럽게 이어져 보인다.
- 원화 금액은 천 단위 구분 기호와 `원` 단위를 포함한다.
- Home 화면은 `screens -> features -> entities -> shared` 의존 방향을 지킨다.
- 화면 spec에 정의된 entity DTO를 screens 계층에서 재선언하지 않는다.
