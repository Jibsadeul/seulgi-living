# 로그인 화면 명세

## 목적

로그인 화면은 미인증 사용자가 앱을 사용하기 전에 반드시 거치는 첫 화면이다.

## 참고 디자인

- `local/design/first_screen.png`

## 범위

### screens/members

- 전체 화면 로그인 UI를 구성한다.
- `member-login` feature의 카카오 로그인 동작을 호출한다.
- 로그인 성공 후 member 기본 정보 완성 여부에 따라 화면을 이동한다.
- 로그인 중/실패/취소 상태를 화면에 반영한다.

## 레이아웃

- 화면 전체 배경은 흰색 기준이다.
- 중앙에는 로고 영역을 배치한다.
- 로고 이미지는 실제 asset이 준비되기 전까지 고정 크기의 placeholder 영역을 사용할 수 있다.
- 하단에는 카카오 로그인 버튼을 고정 배치한다.
- 버튼은 안전 영역을 고려해 하단에 붙되 화면 가장자리와 여백을 둔다.

## 카카오 로그인 버튼

- 텍스트는 `카카오 로그인`을 사용한다.
- 배경색은 카카오 로그인임을 인지할 수 있는 노란색 계열을 사용한다.
- 로그인 진행 중에는 버튼을 비활성화한다.
- 로그인 진행 중 중복 탭으로 로그인 요청이 여러 번 실행되지 않아야 한다.

## 진입 조건

다음 경우 로그인 화면으로 이동한다.

- SecureStore에 토큰이 없다.
- 자동 로그인 확인 중 refresh가 실패했다.
- 인증이 필요한 API에서 refresh 후에도 401이 발생했다.

## 성공 후 이동

- `member.isBasicInfoComplete === true`: 홈 화면
- `member.isBasicInfoComplete === false`: 기본 정보 입력 화면

로그인 화면은 성공 후 뒤로가기로 다시 접근되지 않도록 replace 방식 이동을 사용한다.

## 오류/취소 상태

| 상태                          | 처리                     |
| ----------------------------- | ------------------------ |
| 사용자가 카카오 로그인을 취소 | 로그인 화면 유지         |
| 네트워크 오류                 | 오류 안내 후 재시도 가능 |
| 카카오 인증 실패              | 오류 안내 후 재시도 가능 |
| 서버 세션 발급 실패           | 오류 안내 후 재시도 가능 |

## 검증 기준

- 미인증 상태에서 앱 첫 진입 시 로그인 화면이 보인다.
- 카카오 로그인 버튼은 하단 고정 위치에 표시된다.
- 로그인 중 버튼이 비활성화된다.
- 로그인 성공 후 기본 정보 완성 여부로 화면이 분기된다.
- 로그인 실패 또는 취소 시 로그인 화면에 머문다.
