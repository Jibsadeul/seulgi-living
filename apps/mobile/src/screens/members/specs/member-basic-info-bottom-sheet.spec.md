# members 기본 정보 바텀시트 화면 명세

## 목적

members 기본 정보 바텀시트는 사용자의 서비스 필수 기본 정보를 입력하거나 수정하는 화면 조립 단위이다.

바텀시트는 다음 두 경우에 열린다.

- 카카오 로그인 완료 후 서비스 홈 진입 전
- 홈 화면 상단 설정 버튼 클릭 시

이 화면 spec은 `screens/members`가 담당할 조립, 진입 조건, 닫힘 정책, 화면 상태를 정의한다. 입력 검증과 저장 플로우의 상세 동작은 `features/member-info` spec을 따른다. 사용자와 지역 데이터 구조는 `entities/members` spec을 따른다.

## 범위

### screens/members

- `screens/members`는 기본 정보 바텀시트의 표시 여부와 모드를 결정한다.
- 바텀시트 내부 폼은 `features/member-info`의 UI와 상태를 조합한다.
- 사용자 정보, 닉네임 중복 확인, 지역 목록, 저장 API의 DTO를 화면 계층에서 재선언하지 않는다.
- 화면 계층은 `screens -> features -> entities -> shared` 의존 방향을 지킨다.

## 모드

| 모드         | 진입 지점          | 목적                               |
| ------------ | ------------------ | ---------------------------------- |
| `onboarding` | 카카오 로그인 직후 | 서비스 시작 전 필수 기본 정보 입력 |
| `edit`       | 홈 화면 설정 버튼  | 기존 기본 정보 수정                |

### onboarding 모드

- 카카오 로그인 성공 후 현재 사용자 기본 정보를 조회한다.
- 닉네임, 생년월일, 거주지 중 하나라도 비어 있으면 바텀시트를 연다.
- 필수 입력이 완료되기 전까지 홈 서비스 화면을 사용할 수 없게 한다.
- 닫기 버튼, 바깥 영역 탭, 뒤로가기 동작으로 바텀시트를 닫을 수 없다.
- 저장 성공 시 홈 화면으로 진입한다.

### edit 모드

- 홈 화면의 설정 버튼을 누르면 바텀시트를 연다.
- 현재 사용자 기본 정보를 기본값으로 채운다.
- 사용자가 값을 지워 폼이 유효하지 않은 상태가 되면 저장할 수 없다.
- 필수값 중 빈칸이 있는 상태에서 닫기를 요청하면 저장하지 않고 닫는다.
- 필수값이 모두 있고 Zustand에 저장된 사용자 기본 정보와 현재 폼 값이 같으면 저장 확인 없이 닫는다.
- 필수값이 모두 있고 Zustand에 저장된 사용자 기본 정보와 현재 폼 값이 다르면 `저장하시겠습니까?` 확인 알림을 띄운다.
- Expo Web에서는 `react-native-web`의 `Alert.alert`가 동작하지 않으므로 브라우저 confirm으로 동일한 확인을 처리한다.
- 저장 확인에서 아니요를 선택하면 저장하지 않고 바텀시트를 닫는다.
- 저장 확인에서 저장하면 저장 API를 호출한다.
- 저장 성공 시 바텀시트를 닫고 홈 화면에 최신 사용자 정보를 반영한다.
- 저장 성공 시 `shared/ui/Toast` 공통 UI로 `저장되었습니다.` 토스트를 표시한다.
- 저장 실패 시 바텀시트를 유지하고 실패 메시지를 표시한다.

## 화면 구성

바텀시트에는 다음 영역을 포함한다.

- 제목 영역
  - onboarding: 기본 정보를 입력하라는 의미의 제목
  - edit: 기본 정보를 수정한다는 의미의 제목
- 닉네임 입력 영역
  - 텍스트 입력
  - 중복 확인 버튼
  - 검증 또는 중복 확인 결과 메시지
- 생년월일 입력 영역
  - 년 셀렉트 박스
  - 월 셀렉트 박스
  - 일 셀렉트 박스
- 거주지 입력 영역
  - 시/도 셀렉트 박스
  - 시/군/구 셀렉트 박스
- 저장 버튼
- 구분선 (edit 모드 only)
- 로그아웃 버튼 (edit 모드 only)
- 회원탈퇴 버튼 (edit 모드 only)

## 입력 상태

| 상태           | 화면 요구사항                                                        |
| -------------- | -------------------------------------------------------------------- |
| 초기 로딩      | 사용자 기본 정보와 시/도 목록을 불러오는 동안 폼 조작을 막는다.      |
| 입력 가능      | 모든 필드를 조작할 수 있다. 단, 시/군/구는 시/도 선택 후 활성화한다. |
| 닉네임 확인 중 | 닉네임 중복 확인 버튼 또는 저장 버튼의 중복 요청을 막는다.           |
| 지역 조회 중   | 시/군/구 셀렉트 박스에 로딩 상태를 표시한다.                         |
| 저장 중        | 저장 버튼을 비활성화하고 중복 제출을 막는다.                         |
| 저장 실패      | 바텀시트를 유지하고 실패 메시지를 표시한다.                          |

## 저장 버튼 활성화 기준

다음 조건을 모두 만족할 때만 저장 버튼을 활성화한다.

- 닉네임이 2자 이상 100자 미만이다.
- 닉네임 중복 확인이 완료되었고 사용 가능 상태이다.
- 년, 월, 일이 모두 선택되었다.
- 조합된 생년월일이 유효한 날짜이다.
- 시/도가 선택되었다.
- 선택된 시/도에 속한 시/군/구가 선택되었다.
- 저장 요청이 진행 중이 아니다.

## 닫힘 정책

- onboarding 모드에서는 저장 성공 전까지 닫을 수 없다.
- edit 모드에서는 필수값 중 빈칸이 있으면 저장하지 않고 닫는다.
- edit 모드에서는 필수값이 모두 있고 변경 사항이 없으면 저장하지 않고 닫는다.
- edit 모드에서는 필수값이 모두 있고 변경 사항이 있으면 저장 확인 알림을 띄운다.
- 저장 확인에서 아니요를 선택하면 저장 API 호출 없이 닫는다.
- 닫기 버튼, 바깥 영역 탭, 시스템 뒤로가기는 동일한 닫힘 정책을 따른다.
- 변경 여부는 `entities/members`의 Zustand store에 저장된 `nickname`, `birthday`, `sidoId`, `sigunguId`를 기준으로 판단한다.

## 로그아웃

- edit 모드 바텀시트에서만 노출한다.
- 로그아웃 버튼을 누르면 `로그아웃 하시겠습니까?` 확인 알림을 띄운다.
- 확인 시 `DELETE /api/auth/logout`을 호출한다.
- API 호출 성공/실패 무관하게 SecureStore의 토큰을 삭제하고 로그인 화면으로 이동한다.
- Zustand member store를 초기화한다.

## 회원탈퇴

- edit 모드 바텀시트에서만 노출한다.
- 회원탈퇴 버튼을 누르면 `탈퇴하시겠습니까? 모든 데이터가 삭제됩니다.` 확인 알림을 띄운다.
- 확인 시 `DELETE /api/auth/withdraw`를 호출한다.
- API 호출 성공 시 SecureStore의 토큰을 삭제하고 로그인 화면으로 이동한다.
- API 호출 실패 시 바텀시트를 유지하고 실패 알림을 표시한다.
- Zustand member store를 초기화한다.

## 홈 화면 연동

- 홈 화면 설정 버튼은 `edit` 모드로 members 기본 정보 바텀시트를 연다.
- 홈 화면은 사용자 닉네임을 표시하므로 저장 성공 후 최신 사용자 정보를 다시 조회하거나 캐시를 갱신한다.

## 검증 기준

- 카카오 로그인 직후 필수 기본 정보가 비어 있으면 바텀시트가 열린다.
- onboarding 모드에서 필수값을 채우기 전에는 바텀시트를 닫을 수 없다.
- 홈 설정 버튼을 누르면 edit 모드 바텀시트가 열린다.
- edit 모드에서는 기존 사용자 기본 정보가 입력값으로 표시된다.
- 유효하지 않은 폼 상태에서는 저장 버튼이 비활성화된다.
- edit 모드에서 필수값을 비운 뒤 닫으면 저장 API 호출 없이 닫힌다.
- edit 모드에서 필수값이 모두 있고 변경 사항이 없으면 알림 없이 닫힌다.
- edit 모드에서 필수값이 모두 있고 변경 사항이 있으면 저장 확인 알림이 표시된다.
- 저장 확인에서 아니요를 선택하면 저장 API 호출 없이 닫힌다.
- 저장 확인에서 저장하면 저장 성공 토스트가 표시된다.
- 저장 성공 후 모드에 맞는 후속 동작이 실행된다.
- edit 모드에서 로그아웃/회원탈퇴 버튼이 저장 버튼 아래 구분선 이후에 노출된다.
- onboarding 모드에서 로그아웃/회원탈퇴 버튼이 노출되지 않는다.
- 로그아웃 확인 후 토큰이 삭제되고 로그인 화면으로 이동한다.
- 회원탈퇴 API 성공 후 토큰이 삭제되고 로그인 화면으로 이동한다.
- 회원탈퇴 API 실패 시 바텀시트가 유지되고 실패 알림이 표시된다.
