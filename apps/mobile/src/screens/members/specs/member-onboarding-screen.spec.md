# 신규 회원 기본 정보 화면 명세

## 목적

신규 회원 기본 정보 화면은 카카오 로그인은 완료했지만 서비스 필수 기본 정보가 없는 사용자가 홈 진입 전에 반드시 작성해야 하는 전체 화면이다.

## 참고 디자인

- `local/design/survey.png`

## 범위

### screens/members

- 전체 화면 기본 정보 입력 화면을 구성한다.
- 기존 `features/member-info`의 입력, 검증, 저장 로직을 재사용한다.
- 온보딩 모드에서는 닫기와 건너뛰기를 허용하지 않는다.
- 저장 성공 후 홈 화면으로 이동한다.

## 입력 항목

| 항목     | 필수 | 설명                                      |
| -------- | ---- | ----------------------------------------- |
| 닉네임   | 예   | 2자 이상 100자 미만, 중복 확인 필요       |
| 생년월일 | 예   | 년/월/일 선택 후 `YYYY-MM-DD`로 저장      |
| 거주지   | 예   | 시/도와 시/군/구 선택, 저장은 `sigunguId` |

## 레이아웃

- 화면 전체 배경은 `survey.png` 기준의 연한 배경색을 사용한다.
- 입력 폼 영역은 상단 안전 영역 아래에 배치한다.
- 하단에는 저장 버튼을 고정 배치한다.
- 저장 버튼은 모든 필수 입력과 검증이 완료되기 전까지 비활성화한다.

## 기존 feature 재사용

`features/member-info`가 담당하는 것:

- 닉네임 입력 상태
- 닉네임 중복 확인
- 생년월일 선택 상태
- 시/도, 시/군/구 선택 상태
- 저장 가능 여부 계산
- `PATCH /api/users/me` 저장 요청

screen이 담당하는 것:

- 온보딩 전체 화면 배치
- 닫기/건너뛰기 차단
- 저장 성공 후 홈 이동
- 저장 성공 토스트 또는 안내 표시 여부 결정

## 진입 조건

다음 경우 기본 정보 화면으로 이동한다.

- 카카오 로그인 성공 후 `member.isBasicInfoComplete === false`
- 자동 로그인 성공 후 `/api/users/me`의 `isBasicInfoComplete === false`

## 저장 성공

1. 사용자가 필수 기본 정보를 모두 입력한다.
2. 닉네임 중복 확인을 완료한다.
3. 저장 버튼을 누른다.
4. `PATCH /api/users/me`가 성공한다.
5. 응답 member로 members store를 갱신한다.
6. `isBasicInfoComplete === true`이면 홈 화면으로 replace 이동한다.

## 차단 정책

- 뒤로가기, 닫기, 건너뛰기로 홈 화면에 진입할 수 없다.
- Android back 버튼을 누르면 화면을 벗어나지 않는다.
- 로그아웃 기능은 이번 범위에 없으므로 별도 이탈 경로를 제공하지 않는다.

## 오류 상태

| 조건                    | 처리                              |
| ----------------------- | --------------------------------- |
| 필수값 미입력           | 저장 버튼 비활성화                |
| 닉네임 중복 확인 미완료 | 저장 버튼 비활성화                |
| 닉네임 중복             | 폼 오류 표시                      |
| 지역 목록 조회 실패     | 해당 선택 UI 오류 표시, 저장 차단 |
| 저장 실패               | 오류 안내 후 화면 유지            |

## 검증 기준

- 기본 정보 미완성 회원은 홈 화면에 진입할 수 없다.
- 모든 필수값과 닉네임 중복 확인이 완료되어야 저장할 수 있다.
- 저장 성공 후 홈 화면으로 이동한다.
- Android back 버튼으로 온보딩 화면을 벗어날 수 없다.
- 기존 `member-info` feature의 검증 규칙과 충돌하지 않는다.
