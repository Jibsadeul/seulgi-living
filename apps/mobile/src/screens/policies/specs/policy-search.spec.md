# 청년 정책 검색 화면 명세

## 목적

키워드 검색과 필터링을 통해 사용자가 원하는 정책을 직접 찾을 수 있도록 하는 화면이다.
메인 화면의 검색창 또는 빠른 탐색에서 진입한다.

참고 디자인: Figma node `548:1510`

---

## 화면 구조 — 진입 화면 / 결과 화면 분리

진입 상태와 결과 상태는 **서로 다른 라우트의 별개 화면**이다 (탭바 노출 여부가 다르기 때문).

| 화면           | 라우트                     | 라우트 그룹                                   | 탭바 |
| -------------- | -------------------------- | --------------------------------------------- | ---- |
| 검색 진입 화면 | `/(stack)/policies/search` | `(stack)`                                     | 없음 |
| 검색 결과 화면 | `/(tabs)/policies-results` | `(tabs)` (탭 아이콘으로는 숨김, `href: null`) | 있음 |

> 검색을 막 시작하는 입력 단계는 다른 탭으로 새는 걸 막기 위해 탭바 없이 전체화면으로 띄우고, 결과를 보고 둘러보는 단계부터는 탭바를 다시 보여줘 다른 탭으로 자유롭게 이동할 수 있게 한다 (쿠팡 등 주요 앱의 검색 패턴 참고).

### 1. 검색 진입 화면 (`PoliciesSearchScreen`)

- 검색창 + 최근 검색어 리스트만 표시한다.
- 메인 검색창 탭으로 진입 시 키보드 자동 포커스.
- 키워드 제출 또는 최근 검색어 탭 → `router.replace('/(tabs)/policies-results', { keyword })`로 결과 화면으로 이동.

**최근 검색어**

- 기기 로컬(AsyncStorage)에 최대 10개까지 저장한다. 10개 초과 시 가장 오래된 항목을 제거한다(FIFO).
- 헤더: `최근 검색어` + `모두 지우기` 버튼
- 항목 탭 시 그 키워드로 즉시 검색을 실행한다 (결과 화면으로 이동).
- 항목별 개별 삭제(X) 버튼 제공.

### 2. 검색 결과 화면 (`PoliciesSearchResultScreen`)

- 메인 화면의 빠른 탐색 카테고리/마감임박 탭 → 진입 화면을 거치지 않고 바로 이 화면으로 진입 (키보드 포커스 없음).
- 상단: 검색창(여기서도 키워드 재입력/수정 가능) + 필터 칩 4개 (카테고리 / 지역 / 지원유형 / 정렬) + 검색 결과 개수
- 칩 탭 → 필터 바텀시트가 해당 섹션이 펼쳐진 상태로 오픈
- 결과: 무한스크롤 리스트, `PolicySearchResultCard` 사용 (카테고리·D-day 뱃지, 정책명, 키워드, 연령, 지역 표시)

**Empty State (검색 결과 0건)**

- "조건에 맞는 정책이 없어요" 류 안내 문구

## 필터

필터는 바텀시트(`@gorhom/bottom-sheet`) 하나에 4개 섹션으로 통합되어 있다. 메인 화면의 4개 칩(카테고리/지역/지원유형/정렬) 중 어느 것을 눌러도 같은 바텀시트가 열리며, 누른 칩에 해당하는 섹션만 펼쳐진 상태로 시작한다.

> 처음엔 "정렬"만 바텀시트 없이 칩 자체를 탭해 순환시키는 토글로 분리해뒀으나, 나머지 3개(카테고리/지역/지원유형)와 동작 방식이 달라 "이 칩은 왜 다르게 동작하냐"는 혼란을 유발했다. 일관성을 위해 4개 섹션 모두 같은 바텀시트로 통합했다.

### 카테고리

- 옵션: 주거 / 금융 / 일자리 / 복지 / 교육 / 문화 / 참여 (**복수 선택**)
- `largeCategory`는 "금융·복지·문화"처럼 복합 카테고리 문자열로 저장되어 있어, 서버에서 부분 일치(`contains`)로 매칭한다.
- 여러 개 선택 시 그중 하나라도 포함되면 매칭(OR). 쿼리 파라미터는 쉼표로 join(`largeCategory=주거,금융`).

### 지역

- 옵션: 시/도 목록 (API `/api/regions/sido` 조회, **복수 선택**)
- 선택한 시/도의 `id`들을 쉼표로 join해 `zipCd` 쿼리 파라미터로 전달한다. 서버에서는 그중 하나라도 포함되면 매칭(OR).

### 지원유형

- 옵션: `keywords` 필드에서 추출한 distinct 값 17개 (교육지원/벤처/장기미취업청년/보조금/육아/출산/중소기업/맞춤형상담서비스/인턴/해외진출/주거지원/청년가장/신용회복/바우처/대출/금리혜택/공공임대주택), **복수 선택**
- 서버에서 `keywords` 필드 부분 일치(`contains`)로 매칭한다. 여러 개 선택 시 OR.

### 정렬

- 옵션: 인기순(기본) / 마감기한순 / 상시 (단일 선택)
- 라벨이 곧 정렬 기준이다. 별도의 "정렬 옵션" UI를 따로 두지 않고, 이 섹션 하나로 필터와 정렬을 동시에 결정한다.

### 공통 동작

- 바텀시트 내에서 선택 후 **"적용" 버튼**을 눌러야 결과에 반영된다 (즉시 반영 아님).
- "전체 해제" 버튼으로 4개 섹션의 선택값을 한 번에 초기화할 수 있다.
- 필터 칩을 한 번도 누르지 않았으면 바텀시트 컴포넌트 자체를 마운트하지 않는다 (닫힌 상태에서도 시트 일부가 화면에 살짝 보이는 라이브러리 이슈 회피).
- 카테고리/지역/지원유형 칩에 선택값이 여러 개면 메인 칩 라벨은 `{첫 번째 값} 외 {N}`으로 표시한다.
- 각 섹션의 뱃지는 선택된 개수를 숫자로 표시한다. "정렬"은 항상 하나가 선택된 상태이므로 뱃지를 항상 표시한다(기본값 인기순 포함).

## 정렬 규칙

| 정렬 섹션 값                | 정렬 기준          | 비고                                             |
| --------------------------- | ------------------ | ------------------------------------------------ |
| 인기순 (기본)               | `viewCount` DESC   | 필터 없음 / 키워드 검색 시 기본값                |
| 마감기한순                  | `applyEndDate` ASC | `applyPeriodType=0057001` 필터 동시 적용         |
| 상시                        | `viewCount` DESC   | `applyPeriodType=0057002` 필터 동시 적용         |
| (빠른 탐색 "마감임박" 진입) | `applyEndDate` ASC | `deadlineOnly=true` — 정렬 섹션과 별개 진입 경로 |

- 사용자가 별도의 "정렬" UI를 거치지 않아도, 위 정렬 섹션 선택값에 따라 서버에서 자동으로 정렬 기준이 결정된다.
- "마감기한순"과 빠른 탐색의 "마감임박"은 서로 다른 필터(`applyPeriodType` vs `deadlineOnly`)지만 정렬 기준은 동일하다.
- 페이지네이션: 무한스크롤 (`page`/`limit` 파라미터, `policyListQuerySchema` 그대로 사용)

## 지역 표시 (결과 카드)

- `Policy.region` 필드로 내려온다. BFF가 결과에 등장하는 모든 `zipCd`를 모아 `Sigungu` 테이블을 일괄 조회한 뒤 매핑한다 (N+1 방지).
- 표시 규칙: `noZipLimit=true` → `전국` / 코드 1개 → 해당 시군구명 / 코드 여러 개 → `{첫 번째 지역명} 외 {N}곳`

## 진입 파라미터 매핑

| 진입 경로               | 진입 라우트                | keyword | largeCategory | deadlineOnly |
| ----------------------- | -------------------------- | ------- | ------------- | ------------ |
| 메인 검색창 탭          | `/(stack)/policies/search` | -       | -             | -            |
| 빠른 탐색 카테고리 탭   | `/(tabs)/policies-results` | -       | 있음          | -            |

## 검증 기준

- 최근 검색어는 기기 로컬에만 저장하고 서버로 전송하지 않는다.
- 빠른 탐색으로 진입 시 키보드가 뜨지 않는다.
- 검색 진입 화면은 탭바가 없고, 검색 결과 화면은 탭바가 있다 (서로 다른 라우트 그룹).
- 필터 바텀시트는 "적용" 전까지 결과에 영향을 주지 않는다.
- 카테고리/지역/지원유형/정렬 4개 칩 모두 같은 바텀시트를 열며, 동작 방식이 동일하다.
- 카테고리/지역/지원유형은 복수 선택 가능하며, 서버에서 OR 조건(부분 일치 `contains`)으로 매칭한다.
- 결과 카드에 지역(`region`)이 표시되며, BFF에서 일괄 조회로 N+1을 방지한다.
- API 타입은 `@repo/contract`의 `policyListQuerySchema`/`policyListResponseSchema`를 기준으로 한다.
