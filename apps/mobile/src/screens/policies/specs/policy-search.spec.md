# 청년 정책 검색 화면 명세

## 목적

키워드 검색과 필터링을 통해 사용자가 원하는 정책을 직접 찾을 수 있도록 하는 화면이다.
메인 화면의 검색창 또는 빠른 탐색에서 진입한다.

참고 디자인: Figma node `548:1510`

---

## 화면 상태

### 1. 진입 상태 (키워드 없음 + 필터 없음)

- 검색창 + 최근 검색어 리스트만 표시한다.
- 검색창은 진입 경로에 따라 자동 포커스 여부가 다르다.
  - 메인 검색창 탭으로 진입: 키보드 자동 포커스
  - 빠른 탐색 카테고리 탭으로 진입: 키보드 포커스 안 함 (대신 결과 상태로 바로 진입)

**최근 검색어**

- 기기 로컬(AsyncStorage)에 최대 10개까지 저장한다. 10개 초과 시 가장 오래된 항목을 제거한다(FIFO).
- 헤더: `최근 검색어` + `모두 지우기` 버튼
- 항목 탭 시 그 키워드로 즉시 검색을 실행한다 (결과 상태로 전환).
- 항목별 개별 삭제(X) 버튼 제공.

### 2. 결과 상태 (키워드 입력됨 또는 필터 적용됨)

- 상단: 검색창 + 필터 버튼 (적용된 필터 개수 뱃지 표시)
- 필터 버튼 탭 → 필터 바텀시트 오픈
- 결과: 무한스크롤 리스트, `PolicyCard` 재사용

**Empty State (검색 결과 0건)**

- "조건에 맞는 정책이 없어요" 류 안내 문구
- 빠른 탐색 카테고리를 카드 형태로 재구성하여 추천 노출 (다른 카테고리로 유도)

---

## 필터

### 카테고리 / 지역 / 지원유형 — 바텀시트

- 옵션이 여러 개인 필터는 바텀시트로 노출한다.
- 바텀시트 내에서 선택 후 **"적용" 버튼**을 눌러야 결과에 반영된다 (즉시 반영 아님).

### 기간 — 3단 토글 칩 (바텀시트 없음)

- 칩 탭마다 순환: `전체` → `마감기한순` → `상시` → `전체` ...
- 옵션이 2개뿐이라 모달 없이 가볍게 토글한다.

---

## 정렬 규칙

| 상태                           | 정렬 기준        |
| ------------------------------ | ---------------- |
| 기본 (필터 없음 / 키워드 검색) | `viewCount` DESC |
| 기간 = `마감기한순`            | `daysLeft` ASC   |
| 기간 = `상시`                  | `viewCount` DESC |

- 사용자가 직접 정렬 옵션을 선택하는 UI는 없다. 필터 상태에 따라 서버에서 자동으로 정렬 기준이 결정된다.
- 페이지네이션: 무한스크롤 (`page`/`limit` 파라미터, `policyListQuerySchema` 그대로 사용)

## 진입 파라미터 매핑

| 진입 경로               | keyword | largeCategory | deadlineOnly | 결과 상태 진입 여부 |
| ----------------------- | ------- | ------------- | ------------ | ------------------- |
| 메인 검색창 탭          | -       | -             | -            | 아니오 (진입 상태)  |
| 빠른 탐색 카테고리 탭   | -       | 있음          | -            | 예 (결과 상태)      |
| 빠른 탐색 "마감임박" 탭 | -       | -             | true         | 예 (결과 상태)      |
| 최근 검색어 항목 탭     | 있음    | -             | -            | 예 (결과 상태)      |

---

## 검증 기준

- 최근 검색어는 기기 로컬에만 저장하고 서버로 전송하지 않는다.
- 빠른 탐색으로 진입 시 키보드가 뜨지 않는다.
- 필터 바텀시트는 "적용" 전까지 결과에 영향을 주지 않는다.
- 기간 토글은 바텀시트 없이 칩 탭만으로 동작한다.
- Empty State는 다른 카테고리 탐색을 유도하는 카드를 포함한다.
- API 타입은 `@repo/contract`의 `policyListQuerySchema`를 기준으로 한다.
