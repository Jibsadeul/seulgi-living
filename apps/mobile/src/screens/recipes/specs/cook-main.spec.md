# CookMainScreen

> 레시피 탭(`/recipes`) 진입 화면. Figma 노드 541-1426 "냉장고 메인 (레시피 리스트 최적화 및 스크랩 적용)" 기준.

## 기능 정의

- 레시피 탭의 메인 화면으로, 다음 요소를 위에서 아래로 표시한다.
  1. 헤더: "레시피" 타이틀
  2. 탭 토글: `레시피` / `My 냉장고` (기본값 `레시피`)
  3. 검색바: "오늘 뭐 먹지? 재료나 레시피 검색" placeholder
  4. AI 추천 배너: "냉장고를 구해줘" + "남은 재료로 만드는 최적의 식단 가이드" + "레시피 확인하기" CTA
  5. 상황별 추천 칩: 야식/안주, 초스피드, 다이어트, 건강식 (가로 스크롤)
  6. 모든 레시피 리스트 + "전체보기" 링크: 카드형(썸네일 + 제목 + 한줄설명 + 태그 + 북마크)
  7. 플로팅 버튼: "+ 레시피 입력", 카메라 원형 버튼
- 사용자 인터랙션
  - 탭 토글 클릭 → 레시피 / My 냉장고 콘텐츠 전환 (로컬 상태)
  - 검색바 클릭 → 검색 화면 이동 핸들러(라우트 미존재, TODO)
  - AI 배너 클릭 → `/fridge` (RescueFridgeScreen) 이동
  - 상황별 칩 클릭 → 상황별 리스트 이동 핸들러(라우트 미존재, TODO)
  - 레시피 카드 클릭 → `/recipes/[id]` 이동
  - "전체보기" 클릭 → 전체보기 화면 이동 핸들러(라우트 미존재, TODO)
  - "레시피 입력" 버튼 클릭 → 레시피 등록 이동 핸들러(라우트 미존재, TODO)
  - 카메라 버튼 클릭 → AI 카메라 분석 이동 핸들러(라우트 미존재, TODO)
  - 레시피 카드 북마크 아이콘 클릭 → 로컬 상태로 스크랩 토글(목업)

## 데이터 흐름

- `entities/recipes`, `entities/fridge`가 아직 스텁(`export {}`)이므로, 이번 구현은 `HomeRecipeScrap` 패턴과 동일하게 화면 내부 로컬 MOCK 배열을 사용한다.
  - 추천 레시피 mock, 모든 레시피 mock(각 항목: id, 제목, 설명, 썸네일 placeholder, 태그 배열, 스크랩 여부)
- 실제 API 연동(`@repo/contract` 기반 TanStack Query)은 entities 레이어 구현 이후 별도 작업으로 진행한다.
- 탭 선택 상태, 스크랩 토글 상태는 화면 로컬 `useState`로만 관리한다.

## 특이 사항

- 검색/상황별 리스트/전체보기/레시피 등록/AI 카메라 화면은 `app/` 라우트가 아직 없으므로, 해당 버튼은 자리만 잡고 핸들러는 TODO 주석 + 콘솔 로그로 남긴다.
- 로딩/에러 상태는 mock 데이터 사용으로 해당 없음. `EmptyState` 컴포넌트도 스텁이라 이번 작업에서는 적용하지 않는다.
- 컬러/타이포 토큰은 `apps/mobile/docs/design.md` 팔레트 기준(`main-100`, `surface-default`, `surface-card`, `gray-*`, `tag-*`, `tagText-*`)을 사용하고 Figma 원본 HEX 값을 직접 쓰지 않는다.
- My 냉장고 탭 콘텐츠는 `entities/fridge` 미구현 상태이므로 간단한 안내 텍스트만 표시한다(빈 상태 UI는 추후 별도 spec에서 다룬다).
