# 공유 딥링크 랜딩 페이지 명세

## 목적

앱 외부(카카오톡/문자 등)로 공유된 링크를 받은 사람이, 앱이 설치돼 있으면 해당 화면으로 바로 진입하고, 설치돼 있지 않으면 APK 다운로드로 안내받는 정적 페이지다.

`apps/api`(BFF)나 신규 Next.js 앱을 쓰지 않고, 프레임워크 없는 순수 HTML/JS 정적 파일로 구현한다 — BFF는 API 응답 전용 역할을 유지하고, 페이지 하나를 위해 별도 앱을 만드는 오버헤드도 피한다.

---

## 위치 및 배포

- 경로: `web/share/index.html` (레포 루트, `apps/`·`packages/` 밖의 독립 정적 자산)
- 배포: GitHub Pages, `web/share/` 폴더만 정적 호스팅 (별도 도메인 구매 없음, `https://[github-user].github.io/[repo]/share/` 형태)
- 빌드 도구 없음 — 순수 HTML/CSS/JS 파일 하나로 작성

---

## URL 형식

경로 세그먼트 라우팅(`/share/policies/123`)은 GitHub Pages에서 추가 리다이렉트 설정이 필요해 복잡해지므로, **쿼리 파라미터 방식**을 사용한다.

```
https://[github-user].github.io/[repo]/share/?domain=policies&id=123
```

| 파라미터 | 설명                                  | 예시                  |
| -------- | ------------------------------------- | --------------------- |
| `domain` | 공유 도메인 (커스텀 스킴 경로 prefix) | `policies`, `recipes` |
| `id`     | 해당 도메인 내 리소스 id              | `123`                 |

---

## 동작

1. 페이지 로드 시 `domain`, `id` 쿼리 파라미터를 읽는다.
2. 둘 다 없으면 에러 안내만 표시(잘못된 링크).
3. `window.location.href = \`seulgi-living://${domain}/${id}\`` 호출 (커스텀 스킴 시도).
4. 약 1.5초 타이머 설정 — 그 사이 앱으로 전환되면(브라우저가 백그라운드로 가면) 타이머는 의미 없어지고(탭 종료/이탈), 전환이 안 되면 타이머 종료 후 폴백 UI 노출.
5. 폴백 UI: "앱이 설치되어 있지 않습니다" 안내 문구 + APK 다운로드 버튼(GitHub Releases 링크).

### iOS 처리

- v1은 안드로이드 APK만 지원한다. `navigator.userAgent`로 iOS 감지 시 폴백 UI에 "iOS는 추후 지원" 문구만 표시하고 다운로드 버튼은 숨긴다.

---

## 의존 관계

- APK 다운로드 링크: GitHub Releases에 업로드된 빌드 산출물 URL을 페이지 내 상수로 하드코딩한다(버전 올릴 때마다 수동 갱신, 자동화는 범위 밖).
- 커스텀 스킴 prefix(`seulgi-living`)는 `apps/mobile/app.json`의 `scheme`과 일치해야 한다.
- `domain`/`id` → 모바일 라우트 매핑은 모바일 쪽 `(stack)/{domain}/[id].tsx` 패턴을 따른다고 가정한다(현재 `policies`만 실존, `recipes`는 추후).

---

## 검증 기준

- 쿼리 파라미터 없이 접속 시 에러 안내만 뜨고 스킴 호출을 시도하지 않는다.
- 안드로이드 + 앱 설치 상태에서 접속 시 앱의 해당 상세 화면으로 진입한다(실기기 검증 필요).
- 안드로이드 + 앱 미설치 상태에서 접속 시 1.5초 내 다운로드 버튼이 노출된다.
- iOS 접속 시 다운로드 버튼이 노출되지 않고 안내 문구만 보인다.
